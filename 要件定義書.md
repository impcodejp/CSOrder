# CSOrder システム 要件定義書

**バージョン**: 1.0  
**作成日**: 2026-04-27  
**対象システム**: CSOrder（受注管理システム）

---

## 1. システム概要

### 1.1 目的

営業担当者（社員）の受注データをCSVから取り込み、受注一覧の確認・検索および担当者別の集計帳票を出力する社内向けデスクトップアプリケーション。

### 1.2 システム構成

| 区分 | 技術 | 備考 |
|------|------|------|
| フレームワーク | Tauri 2 | クロスプラットフォームデスクトップアプリ |
| フロントエンド | React 19 + TypeScript + Vite | UIレイヤー |
| バックエンド | Rust | コマンドハンドラ・ビジネスロジック |
| データベース | SQLite | ローカルファイルDB（tauri-plugin-sql） |
| 通信方式 | Tauriコマンド（invoke） | フロント↔バックエンド間IPC |

### 1.3 動作環境

- OS: Windows 10/11
- 画面解像度: 800×600以上
- ネットワーク: 不要（スタンドアロン動作）

## 2. 機能要件

### 2.1 機能一覧

| 機能ID | 機能名 | 概要 |
|--------|--------|------|
| F-01 | データ取込 | CSVファイルを読み込み、受注データをDBに登録する |
| F-02 | 受注一覧 | 登録済み受注データを一覧表示・検索する |
| F-03 | 帳票出力 | 担当者別の受注集計を表示する |
| F-04 | 社員管理 | 社員情報の登録・削除を行う |

---

### 2.2 F-01: データ取込

#### 2.2.1 概要

CSVファイルを指定してプレビューを確認後、受注データをDBに一括登録する。

#### 2.2.2 機能詳細

1. ユーザーがファイル選択ダイアログからCSVファイルを選択する
2. CSVの先頭数行をプレビュー表示する（取込前確認）
3. 「取込開始」ボタン押下でDBへ一括登録を実行する
4. 登録件数を成功メッセージとして表示する
5. エラー発生時はエラーメッセージを表示する（取込中断）

#### 2.2.3 CSVフォーマット

| カラム名 | データ型 | 必須 | 説明 |
|----------|----------|------|------|
| staff_name | 文字列 | ○ | 担当者名 |
| amount | 数値 | ○ | 受注金額（円） |
| gross_profit | 数値 | ○ | 粗利額（円） |
| project_name | 文字列 | ○ | 案件名 |
| client_name | 文字列 | ○ | 顧客名 |

- 1行目はヘッダー行とする
- 文字コード: UTF-8
- 区切り文字: カンマ（,）

#### 2.2.4 バリデーション

- ヘッダー行に必須カラムが全て存在すること
- amount・gross_profit は数値であること
- staff_name・project_name・client_name は空文字でないこと

---

### 2.3 F-02: 受注一覧

#### 2.3.1 概要

登録されている受注データを一覧表示し、担当者名・顧客名で絞り込み検索できる。

#### 2.3.2 機能詳細

1. 画面表示時に全受注データを取得・表示する
2. 担当者名フィルタ（部分一致）で絞り込みができる
3. 顧客名フィルタ（部分一致）で絞り込みができる
4. フィルタ変更時に自動で再検索する
5. 金額・粗利は日本語ロケール形式（1,000,000）で表示する

#### 2.3.3 表示カラム

| カラム | 表示名 | 備考 |
|--------|--------|------|
| id | ID | |
| staff_name | 担当者名 | |
| amount | 受注金額 | 数値フォーマット |
| gross_profit | 粗利 | 数値フォーマット |
| project_name | 案件名 | |
| client_name | 顧客名 | |

---

### 2.4 F-03: 帳票出力

#### 2.4.1 概要

全受注データを担当者別に集計し、件数・合計金額・合計粗利・粗利率を表示する。

#### 2.4.2 機能詳細

1. 担当者ごとに受注データを集計する
2. 集計行の下部に全体合計行を表示する
3. 粗利率は `(粗利合計 / 金額合計) × 100` で算出し、小数第1位まで表示する

#### 2.4.3 表示カラム

| カラム | 表示名 | 算出方法 |
|--------|--------|----------|
| staff_name | 担当者名 | グループキー |
| count | 件数 | COUNT |
| total_amount | 合計金額（円） | SUM(amount) |
| total_gross_profit | 合計粗利（円） | SUM(gross_profit) |
| margin_rate | 粗利率（%） | total_gross_profit / total_amount × 100 |

---

### 2.5 F-04: 社員管理

#### 2.5.1 概要

システムに登録する社員情報（氏名・グレード）を管理する。

#### 2.5.2 機能詳細

1. 登録済み社員を一覧表示する
2. 氏名・グレードを入力して社員を追加できる
3. Enterキーでも追加を実行できる
4. 削除ボタン押下時に確認ダイアログを表示し、承認後に削除する
5. 氏名・グレードの両方が入力されていない場合は追加できない

#### 2.5.3 登録項目

| 項目 | 型 | 必須 | 説明 |
|------|-----|------|------|
| name | 文字列 | ○ | 社員氏名 |
| group | 文字列 | ○ | 所属グループ（例: 営業1部、東京チーム 等） |
| grade | 文字列 | ○ | グレード（例: A, B, マネージャー 等） |

## 3. 非機能要件

### 3.1 パフォーマンス

| 項目 | 要件 |
|------|------|
| CSV取込 | 1,000件以下のCSVを5秒以内に取り込むこと |
| 一覧表示 | 10,000件のデータを3秒以内に表示すること |
| 帳票集計 | 10,000件のデータを3秒以内に集計すること |

### 3.2 信頼性

- CSV取込中にエラーが発生した場合、DBへの書き込みをロールバックすること（トランザクション管理）
- アプリ起動時にDBファイルが存在しない場合、自動でスキーマを作成すること（マイグレーション）

### 3.3 セキュリティ

- SQLインジェクション対策としてプリペアドステートメントを使用すること
- Tauriのケイパビリティ設定により、フロントエンドからアクセスできるコマンドを必要最小限に制限すること

### 3.4 保守性

- バックエンドはコマンド層・サービス層・リポジトリ層に分離すること
- 各層の責務:
  - **command/**: Tauriコマンドのエントリーポイント（フロントからの呼び出し受付）
  - **service/**: ビジネスロジック（集計・バリデーション）
  - **repository/**: DBアクセス（CRUD操作）
  - **model/**: データ構造の定義

## 4. データモデル

### 4.1 ER図（概念）

```
社員 (employees)          受注 (orders)
+----------+              +---------------+
| id (PK)  |              | id (PK)       |
| name     |              | staff_name    |
| grade    |              | amount        |
+----------+              | gross_profit  |
                          | project_name  |
                          | client_name   |
                          | created_at    |
                          +---------------+
```

> ※ 現行設計では受注データの `staff_name` は社員テーブルとの外部キー制約なし。  
> CSVに社員マスタ未登録の担当者名が含まれる場合も取り込み可能とする。

### 4.2 テーブル定義

#### employees テーブル

| カラム名 | 型 | 制約 | 説明 |
|----------|----|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 社員ID |
| name | TEXT | NOT NULL | 社員氏名 |
| group | TEXT | NOT NULL | 所属グループ |
| grade | TEXT | NOT NULL | グレード |

#### orders テーブル

| カラム名 | 型 | 制約 | 説明 |
|----------|----|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 受注ID |
| staff_name | TEXT | NOT NULL | 担当者名 |
| amount | INTEGER | NOT NULL | 受注金額（円） |
| gross_profit | INTEGER | NOT NULL | 粗利額（円） |
| project_name | TEXT | NOT NULL | 案件名 |
| client_name | TEXT | NOT NULL | 顧客名 |
| created_at | TEXT | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 登録日時 |

## 5. バックエンドAPI仕様（Tauriコマンド）

フロントエンドは `invoke(コマンド名, 引数)` でバックエンドを呼び出す。

### 5.1 コマンド一覧

| コマンド名 | 引数 | 戻り値 | 対応機能 |
|------------|------|--------|----------|
| `preview_csv` | `path: String` | `Vec<OrderPreview>` | F-01 |
| `import_orders` | `path: String` | `usize`（取込件数） | F-01 |
| `get_orders` | `staff_name?: String`, `client_name?: String` | `Vec<Order>` | F-02 |
| `get_report` | なし | `Vec<ReportRow>` | F-03 |
| `get_employees` | なし | `Vec<Employee>` | F-04 |
| `add_employee` | `name: String`, `group: String`, `grade: String` | `()` | F-04 |
| `delete_employee` | `id: i64` | `()` | F-04 |

### 5.2 データ型定義（Rust / TypeScript 対応）

#### Order

```rust
// Rust
pub struct Order {
    pub id: i64,
    pub staff_name: String,
    pub amount: i64,
    pub gross_profit: i64,
    pub project_name: String,
    pub client_name: String,
}
```

```typescript
// TypeScript
interface Order {
    id: number;
    staff_name: string;
    amount: number;
    gross_profit: number;
    project_name: string;
    client_name: string;
}
```

#### OrderPreview（取込プレビュー用・id なし）

```rust
pub struct OrderPreview {
    pub staff_name: String,
    pub amount: i64,
    pub gross_profit: i64,
    pub project_name: String,
    pub client_name: String,
}
```

#### ReportRow

```rust
pub struct ReportRow {
    pub staff_name: String,
    pub count: i64,
    pub total_amount: i64,
    pub total_gross_profit: i64,
}
```

> ※ `margin_rate` はフロントエンド側で算出する（`total_gross_profit / total_amount * 100`）

#### Employee

```rust
pub struct Employee {
    pub id: i64,
    pub name: String,
    pub group: String,
    pub grade: String,
}
```

### 5.3 エラーハンドリング

- 全コマンドは `Result<T, String>` を返す
- エラー時はフロントエンドが `catch` し、エラーメッセージとして画面表示する
- DBアクセスエラー、CSVパースエラーは適切なメッセージに変換して返す

## 8. 用語定義

| 用語 | 説明 |
|------|------|
| 受注 | 顧客から受けた発注案件 |
| 担当者 / 社員 | 受注を担当する営業社員 |
| 粗利 | 受注金額から原価を引いた利益額 |
| 粗利率 | 粗利 ÷ 受注金額 × 100（%） |
| グループ | 社員の所属グループ（例: 営業1部、東京チーム） |
| グレード | 社員の等級・職位（例: A, B, マネージャー） |
| CSV取込 | CSVファイルからDBへのデータ一括登録処理 |
