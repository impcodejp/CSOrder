1. システム概要

CSOrder は Tauri v2 + React (TypeScript) + SQLite で構成されるデスクトップアプリケーションです。CS メンバーの受注案件を管理し、担当別・グループ別の売上・粗利・予算達成率を集計・出力することを目的としています。

技術スタック

レイヤー技術
フロントエンドReact 19 + TypeScript + Vite
バックエンドRust (Tauri v2)
DBSQLite (sqlx 0.8 / 自動マイグレーション)
CSV デコードencoding_rs (SJIS → UTF-8)
CSV パースcsv crate (serde Deserialize)
ファイルダイアログtauri-plugin-dialog
ファイル書き込みtauri-plugin-fs
インストーラーNSIS (Windows・日本語・全ユーザーインストール)

budget テーブルは update_employees 実行時に全件削除→再生成される。employees の monthly_budget1〜12 を展開して 12行/人 として保持する。

3-5. 会計月番号の変換ルール

会計月番号 = ((暦月 - 4 + 12) % 12) + 1

例: 4月 → 1, 5月 → 2, 12月 → 9, 3月 → 12
逆変換: 暦月 = ((会計月番号 + 2) % 12) + 1

4-4. service ロジック詳細

employee_service.rs

update_all_employees(pool, employees) の処理フロー:

  1. トランザクション開始
  2. employees テーブルを全件 DELETE
  3. 受け取った社員リストを INSERT
  4. budget テーブルを全件 DELETE
  5. employees から月次予算を展開し budget テーブルへ 12行/人 INSERT
  6. コミット

report_service.rs

画面から受け取る from/to は暦月。DB の order_month_int は会計月番号のため変換してから repository を呼ぶ。

let nfrom = ((from - 4 + 12) % 12) + 1;
let nto   = ((to   - 4 + 12) % 12) + 1;
report_repository::get_report(pool, nfrom, nto).await

csv_import_service.rs の内部関数

関数役割
read_sjis_file(path)ファイルをバイト列で読み込み、encoding_rs で SJIS → UTF-8 変換
validate_row(row, row_num)1行の CsvRow を検証し Ok(NewOrder) または Err(Vec<ValidationError>) を返す
parse_i64(...)文字列を i64 にパース。失敗時はエラーリストに追記して None を返す
require_str(...)空文字チェック。空の場合はエラーリストに追記
date_to_month_int(date)"YYYY/MM/DD" → 会計月番号（((月-4+12)%12)+1）に変換
parse_csv(utf8)全行をループし、契約日空欄行はスキップ、それ以外は validate_row で検証
errors_to_string(errors)エラーリストを "行N: フィールド - メッセージ" 形式の文字列に変換

csv_import_service.rs の公開関数

関数動作
preview_csv(path)parse_csv 実行。エラーがあれば Err。なければ有効行を Vec<CsvPreviewRow> で返す
validate_csv(path)parse_csv 実行。CsvValidationResult（合計/スキップ/有効数/エラーリスト）を返す
import_orders(pool, path)parse_csv 実行。エラーがあれば Err。なければ全行を upsert_orders でDB登録し件数を返す
契約日が空欄の行はエラーではなくスキップ。upsert_orders は INSERT OR REPLACE INTO orders を使用し、project_code の UNIQUE 制約で既存レコードを上書きする。

4-5. report_repository::get_report の SQL

SELECT
    e.name                                                    AS cs_name,
    e."group",
    e.grade,
    COALESCE(COUNT(o.id), 0)                                 AS project_count,
    COALESCE(SUM(o.amount), 0)                               AS amount,
    COALESCE(SUM(o.gross_profit), 0)                         AS gross_profit,
    COALESCE(b_sum.sum_budget, 0)                            AS bugget
FROM employees e
LEFT JOIN orders o
    ON  o.employee_code    = e.employee_code
    AND o.order_month_int BETWEEN ?2 AND ?3
LEFT JOIN budget b_month
    ON  b_month.code      = e.employee_code
    AND b_month.month_int = ?1
LEFT JOIN (
    SELECT code, SUM(budget) AS sum_budget
    FROM   budget
    WHERE  month_int BETWEEN ?2 AND ?3
    GROUP  BY code
) b_sum ON b_sum.code = e.employee_code
GROUP BY e.id, e.name, e."group", e.grade
ORDER BY e."group", e.grade DESC, e.name

バインドパラメータ: ?1 = nto（当月）, ?2 = nfrom（期間開始）, ?3 = nto（期間終了）

返却されるのは指定期間（from〜to月）の合計1セット。フロントエンドは from == to の場合に「4月件数」、異なる場合に「4~6月件数」のようにラベルを切り替える。

8. エラーハンドリング方針

レイヤー方針
repositorysqlx エラーを .to_string() で Err(String) に変換して上位へ伝播
servicerepository の Err をそのまま伝播。ビジネスルール違反も Err(String)
commandResult<T, String> をそのまま Tauri に返す。Tauri が JSON シリアライズ
フロントエンドinvoke の catch ブロックで String(err) を画面表示
