# CSOrder システム 要件定義書

**バージョン**: 1.1  
**作成日**: 2026-04-27  
**最終更新**: 2026-04-30  
**対象システム**: CSOrder（CS受注予実管理システム）

---

## 1. システム概要

### 1.1 目的

CS（カスタマーサクセス）メンバーの受注データをCSVから取り込み、担当者別・グループ別の売上・粗利・予算達成率を管理・集計する社内向けデスクトップアプリケーション。

### 1.2 システム構成

| 区分 | 技術 | 備考 |
|------|------|------|
| フレームワーク | Tauri 2 | クロスプラットフォームデスクトップアプリ |
| フロントエンド | React 19 + TypeScript + Vite | UIレイヤー |
| バックエンド | Rust | コマンドハンドラ・ビジネスロジック |
| データベース | SQLite（sqlx 0.8） | ローカルファイルDB・自動マイグレーション |
| 通信方式 | Tauriコマンド（invoke） | フロント↔バックエンド間IPC |
| 文字コード変換 | encoding_rs | SJIS → UTF-8 変換 |
| CSVパース | csv crate（serde Deserialize） | |
| インストーラー | NSIS | Windows向け・日本語・全ユーザー |

### 1.3 動作環境

- OS: Windows 10/11
- 画面解像度: 1280×720 以上推奨
- ネットワーク: 不要（スタンドアロン動作）

### 2.1 機能一覧

| 機能ID | 機能名 | 概要 |
|--------|--------|------|
| F-01 | データ取込 | SJIS形式CSVを読み込み、受注データをDBに登録する |
| F-02 | 受注一覧 | 登録済み受注データを一覧表示・検索する |

---

### 2.2 F-01: データ取込

#### 2.2.1 概要

SJIS形式のCSVファイルを選択してプレビューを確認後、受注データをDBに一括登録する。

#### 2.2.2 機能詳細

1. ユーザーがファイル選択ダイアログからCSVファイル（.csv）を選択する
2. バックエンドで SJIS → UTF-8 変換後、CSVをパース・バリデーションする
3. プレビューテーブル（担当者名・売上・粗利・案件名・顧客名・契約日）を表示する
4. エラーがある場合はプレビューを表示せずエラー内容を一覧表示する
5. 「取込開始」ボタン押下で全件をDBへ一括登録する（all-or-nothing）
6. 取込件数を成功メッセージとして表示する

#### 2.2.3 CSVフォーマット

- 文字コード: **Shift-JIS**（バックエンドで自動変換）
- 区切り文字: カンマ（,）
- 1行目: ヘッダー行

| カラム名（ヘッダー） | データ型 | 必須 | 説明 |
|---------------------|----------|------|------|
| 案件番号 | 数値 | ○ | 案件を一意に識別する番号（UNIQUE） |
| 案件名 | 文字列 | ○ | 案件名称 |
| 顧客名 | 文字列 | ○ | 顧客企業名 |
| 売上合計 | 数値 | ○ | 受注金額（千円単位） |
| 粗利合計 | 数値 | ○ | 粗利額（千円単位） |
| 契約日 | YYYY/MM/DD | ○ | 契約締結日（空欄行はスキップ） |
| 案件主担当者ID | 数値 | ○ | 社員番号（employees と結合するキー） |
| 案件主担当者名 | 文字列 | ○ | 担当者氏名 |

#### 2.2.4 バリデーション

| フィールド | 検証内容 |
|-----------|---------|
| 案件番号 | 非空・数値変換可能 |
| 案件主担当者ID | 非空・数値変換可能 |
| 案件主担当者名 | 非空 |
| 案件名 | 非空 |
| 顧客名 | 非空 |
| 売上合計・粗利合計 | 数値変換可能 |
| 契約日 | YYYY/MM/DD 形式・月が 1〜12 の範囲内 |

- 契約日が空欄の行は**エラーではなくスキップ**（スキップ件数は表示）
- バリデーションエラーが1件でもある場合はDBへの書き込みを行わない
- 同一案件番号のデータは上書き（UPSERT）する

### 2.3 F-02: 受注一覧

#### 2.3.1 概要

登録されている受注データを一覧表示し、担当者名・顧客名で絞り込み検索できる。

#### 2.3.2 機能詳細

1. 画面表示時に全受注データを取得・表示する
2. 担当者名フィルタ（部分一致）で絞り込みができる
3. 顧客名フィルタ（部分一致）で絞り込みができる
4. フィルタ変更時に自動で再検索する
5. 金額は日本語ロケール形式（¥1,000,000）で表示する

#### 2.3.3 表示カラム

| カラム | 表示名 | 備考 |
|--------|--------|------|
| id | ID | |
| cs_name | 担当者名 | |
| order_date | 契約日 | YYYY/MM/DD |
| amount | 売上合計 | 千円単位・数値フォーマット |
| gross_profit | 粗利合計 | 千円単位・数値フォーマット |
| project_name | 案件名 | |
| client_name | 顧客名 | |

### 2.4 F-03: 帳票出力

#### 2.4.1 概要

指定した期間（月〜月）の受注データを担当者別・グループ別に集計し、売上・粗利・予算・達成率を表示する。CSVエクスポートも可能。

#### 2.4.2 機能詳細

1. 期間（開始月〜終了月）を入力し「更新」ボタンで集計を実行する
2. 「担当別」タブと「グループ集計」タブを切り替えて表示する
3. 担当別タブ: 社員ごとに件数・売上・粗利・予算・達成率を表示し、グループ小計行を挿入する
4. グループ集計タブ: グループごとに人数・件数・売上・粗利・予算・達成率を表示する
5. 各タブに「CSV出力」ボタンがあり、表示内容をCSVファイルに出力できる
6. 達成率は予算に対する粗利の割合（粗利 ÷ 予算 × 100）で算出する
7. 達成率 ≥ 100%: 緑色表示、< 80%: 赤色表示

#### 2.4.3 表示カラム（担当別タブ）

| カラム | 内容 | 算出方法 |
|--------|------|---------|
| 名前 | 担当者名 | |
| GROUP | グループ名 | |
| GRADE | グレード | |
| {期間}件数 | 受注件数 | COUNT |
| {期間}売上 | 売上合計（千円） | SUM(amount) |
| {期間}粗利 | 粗利合計（千円） | SUM(gross_profit) |
| {期間}予算 | 予算合計（千円） | SUM(budget) |
| {期間}達成率 | 予算達成率 | 粗利 ÷ 予算 × 100（%） |

> ※ 単月指定時は「4月件数」、複数月指定時は「4~6月件数」のように表示

#### 2.4.4 CSVエクスポートフォーマット（担当別）

```
名前,GROUP,GRADE,{期間}件数,{期間}売上(千円),{期間}粗利(千円),{期間}予算(千円),{期間}達成率
```

### 2.5 F-04: 社員管理

#### 2.5.1 概要

システムに登録する社員情報と月次予算を管理する。スプレッドシート形式で一括編集し、「更新」で全件保存する。

#### 2.5.2 機能詳細

1. 登録済み社員を一覧表示する（テーブル形式で全項目を表示）
2. 「＋行追加」ボタンで末尾に空行を追加し、手入力で社員を追加できる
3. 各行の「削除」ボタンで行を削除できる
4. 「更新」ボタン押下で全行をDBに一括保存する（全件置換）
5. 「CSV取込」でCSVファイルを選択し、画面に展開する（保存は「更新」で実行）
6. 「CSV出力」で現在の画面内容をCSVファイルに出力できる
7. 「レイアウト出力」でヘッダーのみのCSVテンプレートを出力できる

#### 2.5.3 登録項目

| 項目 | 型 | 必須 | 説明 |
|------|-----|------|------|
| 社員番号 | 数値 | ○ | 社員を一意に識別する番号（1以上）|
| 社員名 | 文字列 | ○ | 社員氏名 |
| グループ | 文字列 | ○ | 所属グループ（例: 営業1部、東京チーム）|
| グレード | 文字列 | ○ | グレード・職位（例: A, B, マネージャー）|
| 4月予算〜3月予算 | 数値 | - | 各月の予算額（千円単位、省略時0） |

#### 2.5.4 社員CSVフォーマット

- 文字コード: UTF-8（BOM付き）
- ヘッダー: `社員番号,社員名,グループ,グレード,4月予算,5月予算,6月予算,7月予算,8月予算,9月予算,10月予算,11月予算,12月予算,1月予算,2月予算,3月予算`
- 月順: 会計年度順（4月〜3月）

#### 2.5.5 保存時バリデーション

- 全行の社員番号が1以上の数値であること
- 全行の社員名・グループ・グレードが非空であること

## 3. 非機能要件

### 3.1 パフォーマンス

| 項目 | 要件 |
|------|------|
| CSV取込 | 1,000件以下のCSVを5秒以内に取り込むこと |
| 一覧表示 | 10,000件のデータを3秒以内に表示すること |
| 帳票集計 | 10,000件のデータを3秒以内に集計すること |

### 3.2 信頼性

- CSV取込中にエラーが発生した場合、DBへの書き込みをロールバックすること（all-or-nothing）
- アプリ起動時にDBファイルが存在しない場合、自動でスキーマを作成すること（自動マイグレーション）
- 社員情報更新時（`update_employees`）はトランザクションで employees・budget テーブルを一括更新する

### 3.3 セキュリティ

- SQLインジェクション対策としてプリペアドステートメント（sqlx）を使用すること
- Tauriのケイパビリティ設定により、フロントエンドからアクセスできるコマンドを必要最小限に制限すること

### 3.4 保守性

- バックエンドはコマンド層・サービス層・リポジトリ層に分離すること
- 各層の責務:
  - **command/**: Tauriコマンドのエントリーポイント（引数受け取り・AppState受け渡しのみ）
  - **service/**: ビジネスロジック（集計・バリデーション・変換処理）
  - **repository/**: DBアクセス（CRUD・SQLクエリ）
  - **model/**: 各層で共有するデータ構造の定義

## 4. データモデル

### 4.1 ER図（概念）

```
社員 (employees)                 受注 (orders)
+--------------------+           +---------------------+
| id (PK)            |           | id (PK)             |
| employee_code (UQ) |←──────────| employee_code       |
| name               |           | project_code (UQ)   |
| group              |           | cs_name             |
| grade              |           | amount              |
| monthly_budget1〜12|           | gross_profit        |
+--------------------+           | project_name        |
          |                      | client_name         |
          |                      | order_date          |
          ↓                      | order_month_int     |
予算 (budget)                    +---------------------+
+--------------------+
| id (PK)            |
| code (employee_code)|
| cs_name            |
| month_int          |
| budget             |
+--------------------+
```

> ※ orders.employee_code と employees.employee_code は結合キーだが、DB外部キー制約は設定していない。  
> 社員マスタ未登録の担当者の受注も取り込み可能とする。

### 4.2 テーブル定義

#### employees テーブル

| カラム名 | 型 | 制約 | 説明 |
|----------|----|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 社員ID |
| employee_code | INTEGER | UNIQUE, NOT NULL | 社員番号 |
| name | TEXT | NOT NULL | 社員氏名 |
| group | TEXT | NOT NULL | 所属グループ |
| grade | TEXT | NOT NULL | グレード |
| monthly_budget1 | INTEGER | | 4月予算（千円） |
| monthly_budget2 | INTEGER | | 5月予算（千円） |
| ... | | | |
| monthly_budget12 | INTEGER | | 3月予算（千円） |

#### orders テーブル

| カラム名 | 型 | 制約 | 説明 |
|----------|----|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 受注ID |
| project_code | INTEGER | UNIQUE, NOT NULL | 案件番号 |
| employee_code | INTEGER | NOT NULL | 担当社員番号 |
| cs_name | TEXT | NOT NULL | 担当者名 |
| amount | INTEGER | NOT NULL | 売上合計（千円） |
| gross_profit | INTEGER | NOT NULL | 粗利合計（千円） |
| project_name | TEXT | NOT NULL | 案件名 |
| client_name | TEXT | NOT NULL | 顧客名 |
| order_date | TEXT | NOT NULL | 契約日（YYYY/MM/DD） |
| order_month_int | INTEGER | NOT NULL | 会計月番号（4月=1〜3月=12） |

#### budget テーブル

| カラム名 | 型 | 制約 | 説明 |
|----------|----|------|------|
| id | INTEGER | PK, AUTOINCREMENT | ID |
| code | INTEGER | NOT NULL | employee_code と同値 |
| cs_name | TEXT | NOT NULL | 氏名 |
| month_int | INTEGER | NOT NULL | 会計月番号（1〜12） |
| budget | INTEGER | NOT NULL | 月次予算（千円） |

> `budget` テーブルは `update_employees` 実行時に全件削除→再生成される（employees の monthly_budget1〜12 を12行/人に展開）。

### 4.3 会計月番号の変換ルール

```
会計月番号 = ((暦月 - 4 + 12) % 12) + 1

例: 4月 → 1, 5月 → 2, 12月 → 9, 3月 → 12
```

## 5. バックエンドAPI仕様（Tauriコマンド）

フロントエンドは `invoke(コマンド名, 引数)` でバックエンドを呼び出す。

### 5.1 コマンド一覧

| コマンド名 | 引数 | 戻り値 | 対応機能 |
|------------|------|--------|----------|
| `get_employees` | なし | `Vec<Employee>` | F-04 |
| `update_employees` | `employees: Vec<UpdateEmployee>` | `()` | F-04 |
| `get_orders` | `cs_name?: String`, `client_name?: String` | `Vec<Order>` | F-02 |
| `get_report` | `from: i64`, `to: i64` | `Vec<CsMemberOrderSummary>` | F-03 |
| `preview_csv` | `path: String` | `Vec<CsvPreviewRow>` | F-01 |
| `validate_csv` | `path: String` | `CsvValidationResult` | F-01 |
| `import_orders` | `path: String` | `usize`（取込件数） | F-01 |

- 全コマンドは `Result<T, String>` を返す
- エラー時はフロントエンドが `catch` し、エラーメッセージを画面表示する

### 5.2 主要データ型定義

#### Employee（社員情報）

```typescript
interface Employee {
  id: number;
  employeeCode: number;
  name: string;
  group: string;
  grade: string;
  monthlyBudget1: number;   // 4月
  monthlyBudget2: number;   // 5月
  // ...
  monthlyBudget12: number;  // 3月
}
```

#### Order（受注情報）

```typescript
interface Order {
  id: number;
  csName: string;
  orderDate: string;      // "YYYY/MM/DD"
  amount: number;
  grossProfit: number;
  projectName: string;
  clientName: string;
}
```

#### CsMemberOrderSummary（帳票集計行）

```typescript
interface CsMemberOrderSummary {
  csName: string;
  group: string;
  grade: string;
  projectCount: number;   // 指定期間の受注件数
  amount: number;         // 指定期間の売上合計（千円）
  grossProfit: number;    // 指定期間の粗利合計（千円）
  bugget: number;         // 指定期間の予算合計（千円）
}
```

#### CsvValidationResult（CSVバリデーション結果）

```typescript
interface CsvValidationResult {
  total: number;       // バリデーション対象行数（スキップ除く）
  skipped: number;     // 契約日空欄によるスキップ行数
  validCount: number;  // エラーなし行数
  errors: { row: number; field: string; message: string }[];
}
```

## 7. 用語定義

| 用語 | 説明 |
|------|------|
| 受注 | 顧客から受けた発注案件 |
| 担当者 / 社員 | 受注を担当するCSメンバー |
| 粗利 | 受注金額から原価を引いた利益額 |
| 達成率 | 粗利 ÷ 予算 × 100（%）。予算に対する粗利の達成度 |
| 会計月番号 | 4月始まりの会計年度における月の番号（4月=1, 3月=12） |
| グループ | 社員の所属グループ（例: 1G、2G、営業支援Gなど） |
| グレード | 社員の等級・職位（例:G1、G2、M1など） |
| CSV取込 | CSVファイルからDBへのデータ一括登録処理 |
| UPSERT | 同一案件番号のレコードが存在する場合は上書き、なければ新規挿入 |
